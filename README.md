# QueueCure
QueueCure '26: An integrated, high-performance triage and priority-based queue management system designed for clinical environments. Features real-time tracking, intelligent database switching between local-caching and MongoDB, and an automated Smart Search system for seamless patient history management.

# QueueCure '26

## The Problem
Clinical triage and patient management are often plagued by manual inefficiencies, leading to long wait times and potential errors in high-pressure environments. QueueCure '26 was developed to solve this by providing a streamlined, priority-based digital triage system.

## Key Features
* **Intelligent Triage:** Automated sorting based on severity (Emergency, Urgent, Routine).
* **Dual-Engine Architecture:** Seamlessly switches between local-caching and MongoDB based on server availability.
* **Smart Search:** Advanced retrieval system for active patients and historical visit records.
* **Real-time Notifications:** Audio and visual cues for patient calls, improving lobby communication.
* **Responsive UI:** Built with React and Framer Motion for a modern, high-usability experience.

## Tech Stack
* **Frontend:** React, Tailwind CSS, Framer Motion, Lucide React
* **Backend:** Node.js, Express (with MongoDB integration)
* **Storage:** LocalStorage (Offline mode), MongoDB (Live mode)

## How to Run
1. Clone the repository.
2. Install dependencies: `npm install`
3. Start the application: `npm run dev`

## Why This Project?
This project demonstrates my ability to build robust, fault-tolerant full-stack applications with an emphasis on state management, persistent data, and real-world clinical utility.
